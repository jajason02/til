# 100% 정수 연산 양자화 / TTA / 적응적 센싱

> 2026-03

## 배운 것

### 일반 양자화 vs 100% 정수 연산 양자화

- 일반 양자화: 값(value)을 실수 → 정수로 변환
- 100% 정수 연산 양자화: 연산 과정(process) 자체도 정수만 사용

문제: 양자화 공식 `r = S(q - Z)`에서 S(스케일링 파라미터)가 나눗셈 결과라 항상 실수 → 가속기(NPU) 사용 불가.

**스케일링 파라미터 공식**

```
S = (r_max - r_min) / (2^n - 1)    ← 실수
Z = floor(-r_min / S)               ← 정수
```

---

### 100% 정수 연산 기법 3가지

**기법 1: 실수 파트 묶기 (Grouping)**

행렬 곱에서 여러 S 값을 하나의 M으로 묶음.

```
M = S1 × S2 / S3    (실수들을 하나로 합침)
나머지는 전부 정수 → M만 해결하면 됨
```

**기법 2: 비트 시프팅 (Bit Shifting)**

M이 0~1 사이라 정수 양자화 불가 → 뻥튀기 후 양자화.

```
1. 스케일 업: M × 2^n  (왼쪽 시프트)
2. 양자화: floor()로 정수 변환
3. 연산 수행 후 스케일 다운: >> n  (오른쪽 시프트)

예) M = 0.34, n = 8
0.34 × 256 = 87.04 → floor → 87
사용 시: 87 >> 8
```

**기법 3: BN 폴딩 (Batch Normalization Folding)**

BatchNorm의 나눗셈/루트 연산을 앞뒤 레이어 가중치에 흡수.

```
W_fold = (γ × W) / σ
b_fold = β - (γ × μ) / σ
```

BN 레이어가 사라지고 W_fold, b_fold만 양자화하면 됨.

---

### 스케일링 법칙과 분포 이동

**스케일링 법칙**: 모델 크기 + 학습 데이터량 + 컴퓨팅이 늘면 성능도 비례 향상. 하지만 비용 폭발.

**분포 이동 (Distribution Shift)**: 학습 데이터와 테스트 데이터의 분포가 달라지는 현상.

대응 전략:
- 초거대 AI: 지식 영역 확대 (비용 폭발)
- 적응적 센싱: 테스트 분포를 학습 분포로 끌어당김

---

### TTA (Test-Time Adaptation)

라벨 없는 테스트 데이터로 모델을 실시간 적응. 라벨이 없으므로 엔트로피를 손실 함수로 사용.

**TENT**: BatchNorm의 μ, σ는 테스트 데이터로 즉시 업데이트. γ, β는 엔트로피 최소화로 역전파.

한계: 이상치/노이즈도 모두 신뢰하여 불안정.

**SAR**: TENT의 불안정성 개선.
- 샘플 필터링: 엔트로피 높은 샘플(이상치)은 학습에서 제외
- 손실 평탄화: 이전 값과 차이가 너무 크면 억제

---

### VLM 기반 TTA

**TPT (Test-time Prompt Tuning)**

```
이미지 1장 → 27개 증강 → 엔트로피 낮은 상위 6개 선택 → 텍스트 프롬프트만 업데이트
(모델 파라미터 동결)
```

**PromptAlign**: TPT + 이미지 인코더 프롬프트도 추가 튜닝. 각 레이어별 평균/분산을 소스 도메인과 정렬.

| 기법 | 텍스트 프롬프트 | 이미지 프롬프트 |
|:---|:---|:---|
| TPT | 업데이트 | 고정 |
| PromptAlign | 업데이트 | 업데이트 |

### 적응적 센싱 (Adaptive Sensing)

> "눈이 나쁘면 공부를 더 하는 게 아니라 안경을 쓴다"

모델을 바꾸는 게 아니라 센서가 AI 친화적 데이터를 제공해 분포 이동 자체를 억제.

**LENS**: 동일 피사체를 다양한 카메라 파라미터로 촬영 → 컨피던스 가장 높은 이미지 선택 → 최종 예측.

결과: 파라미터 50배 작은 모델 + LENS → 초거대 모델 대비 4~10% 정확도 향상.

---

### 도메인 지식 주입

전문 지식을 모델에 직접 반영 → 경량 모델로 높은 성능 가능.

수면 의학 AI 사례:
- 수면 채점을 시계열 → 컴퓨터 비전 문제로 전환 (파형 스크린샷)
- 에폭 간 트랜스포머로 전후 관계 모델링
- 무호흡 직접 감지 대신 호흡 각성 감지 후 역추적 → 실시간 처리 달성

## 막혔던 것 / 해결

BN 폴딩에서 왜 BN이 사라지는지 이해 안 됐음.  
BN의 나눗셈(σ)과 뺄셈(μ) 연산을 앞 레이어 W와 b에 미리 계산해서 합쳐버리는 것.  
추론 시 BN을 별도로 실행할 필요가 없어짐.
