## 배운 것

### 수 체계

**부호 크기 표현 (Sign-Magnitude)**: MSB = 부호비트. 단점: +0과 -0이 두 가지 패턴 → 덧셈 연산 그대로 사용 불가.

**2의 보수 (Two's Complement)**: 현재 컴퓨터에서 실제로 사용. MSB에 -2^(n-1) 가중치 부여.

```
n비트 표현 범위: -2^(n-1) ~ 2^(n-1)-1
4비트: -8 ~ +7 / 8비트: -128 ~ +127

부호 반전: 비트 전부 반전 → +1
+3 = 0011 → 반전 1100 → +1 → 1101 = -3
```

오버플로우: 양수+양수=음수 또는 음수+음수=양수이면 오버플로우.

비트 확장:
- 부호 확장(Sign Extension): MSB 복사 → 값 유지
- 제로 확장(Zero Extension): 0으로 채움 → 패턴 유지

### 고정 소수점 (Fixed-Point)

`[정수부 k비트 | 소수부 m비트]` 형식. 소수점 위치는 저장하지 않고 프로그래머 머릿속에만 존재.

| 연산 | 처리 |
|:---|:---|
| 덧셈/뺄셈 | 그대로 수행 가능 |
| 곱셈 | 결과를 2^m으로 나눠야 함 (오른쪽 시프트) |
| 나눗셈 | 결과를 2^m으로 곱해야 함 (왼쪽 시프트) |

```
예) 4.4 고정소수점에서 1.5 × 0.5
1.5 → 내부값 24, 0.5 → 내부값 8
24 × 8 = 192 → 192 >> 4 = 12 → 소수점 찍으면 0.75 ✓
```

### 부동 소수점 (Floating-Point)

값 = (-1)^S × M × β^E

**IEEE 754 단정밀도 (32비트)**: `[S(1) | Exponent(8) | Fraction(23)]`, 바이어스 127.

히든 비트: 정규화 수의 앞 1은 저장하지 않음 → 유효숫자 1비트 추가.

특수값:

| 지수부 | 가수부 | 의미 |
|:---|:---|:---|
| 00000000 | 0 | 0 |
| 11111111 | 0 | ±무한대 |
| 11111111 | ≠0 | NaN |

정규화 이유: 표현 중복 방지 + 유효숫자 최대 활용.

바이어스 지수 이유: 지수를 양수 영역으로 이동 → 대소 비교를 정수 비교와 동일하게 처리.

### 모델 경량화

**양자화 (Quantization)**: 32bit Float → 8/4bit 정수로 변환.

| 구분 | QAT | PTQ |
|:---|:---|:---|
| 시점 | 학습 중 | 학습 후 |
| 정확도 | 높음 | 상대적으로 낮음 |
| LLM 적합성 | 낮음 | 높음 (학습 데이터 미공개) |

BF16이 FP16보다 딥러닝에 유리한 이유: FP32와 동일한 지수 범위 유지.

**프루닝 (Pruning)**: 중요도 낮은 가중치를 0으로 만들어 생략.
- Unstructured: 임의 위치 제거 (압축률 높지만 GPU 비친화)
- Structured: 행/열 단위 제거 (GPU 친화적)
- GPU는 95% 이상 압축 시에야 속도 이득. 딥러닝 수준(60~80%)으로는 미미.

**지식 증류 (Knowledge Distillation)**: 큰 Teacher 모델이 작은 Student 모델을 가르침.
역량 차이가 너무 크면 중간 Assistant 모델 활용.

### Fine-Tuning 방법 비교

| 방식 | 특징 |
|:---|:---|
| In-Context Learning | 프롬프트만 수정, 비용 없음, 표현력 한계 |
| Full Fine-Tuning | 성능 최고, 비용 폭발, 망각 발생 |
| LoRA | 저랭크 행렬 추가, 효율·성능 균형 |
| QLoRA | 4bit 양자화 + LoRA 결합 |

**LoRA**: 원본 W 고정, A(d×r) + B(r×d) 두 개의 작은 행렬만 학습.

```
W (1000×1000) = 1,000,000개 학습 필요
↓ rank=1 LoRA 적용
A (1000×1) + B (1×1000) = 2,000개만 학습 (0.2%)
```

**QLoRA**: 원본 4bit 양자화(고정) + LoRA 벡터(16bit, 학습) → 4배 압축 + 정확도 복구.

주의: 파인튜닝 ≠ 경량화. 파인튜닝은 특정 도메인 특화가 목적이지 추론 속도 향상이 목적이 아님.

## 막혔던 것 / 해결

IEEE 754 변환에서 바이어스 127을 빼는 걸 더하는 걸로 헷갈렸음.  
저장된 지수(Exponent) - 127 = 실제 지수. 실제 지수 + 127 = 저장값.  
예) 실제 지수 2 → 저장값 129 = 10000001.
