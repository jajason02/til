# 스택 응용 — 후위표기법 (Postfix)

> 2026-04-16

## 배운 것

중위 표기식(`6+5*(2-8)/2`)을 후위 표기식(`6528-*2/+`)으로 변환 후 계산.

### 변환 (Infix → Postfix)

- 숫자: 바로 결과에 추가
- 연산자: 스택 안의 연산자와 우선순위 비교, 자신보다 높거나 같은 게 있으면 pop 후 추가
- `(`: icp=3으로 무조건 push, 하지만 스택 안에서는 isp=0 (뒤에 오는 모든 연산자가 위에 쌓임)
- `)`: `(`를 만날 때까지 pop해서 결과에 추가

```python
icp = {"+": 1, "-": 1, "*": 2, "/": 2, "(": 3}
isp = {"+": 1, "-": 1, "*": 2, "/": 2, "(": 0}
```

### 계산 (Postfix 계산)

- 숫자: 스택에 push
- 연산자: 숫자 2개 pop → 연산 → 결과 push
- **주의**: 먼저 pop한 게 오른쪽 피연산자

```python
right = stack.pop()
left = stack.pop()
result = left - right  # 순서 중요!
```

## 막혔던 것 / 해결

`(`의 icp와 isp가 다른 이유가 처음엔 이해 안 됐음.  
icp=3이라 들어올 땐 무조건 push되고, isp=0이라 안에서는 아무도 pop 안 당함 → 괄호 내부를 독립적으로 처리하기 위한 설계.
