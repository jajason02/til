# 연결리스트 구현 (Node Pool 방식)

> 2026-04-28

## 배운 것

연결리스트는 노드를 동적으로 연결하는 자료구조. 배열과 달리 삽입/삭제가 효율적.

**노드 풀(Node Pool) 방식**: 미리 5000개 노드 공간을 할당해두고, 필요할 때마다 꺼내쓰는 방식. SWEA/코테 같이 빠른 입출력이 필요한 환경에 유리.

**구조:**
- `nodeArr`: 노드 저장소 (5000개 공간 미리 할당)
- `head`: 리스트의 첫 번째 노드
- `tail`: 리스트의 마지막 노드 (끝에 빠르게 추가할 때 사용)
- `nodeCnt`: 현재까지 사용한 노드 개수 (다음 빈 자리 추적)

**핵심 연산:**

```python
# 1. 새 노드 생성 (O(1))
node = lst.getNewNode(data)
# nodeArr[nodeCnt]에 노드 생성, nodeCnt 증가, 노드 반환

# 2. idx 위치에 nums 삽입 (O(n))
lst.insert(idx, [1, 2, 3])
# idx 위치를 찾은 후, 노드들을 하나씩 연결

# 3. idx 위치부터 cnt개 삭제 (O(n))
lst.remove(idx, cnt)
# idx-1 노드와 (idx+cnt) 노드를 직접 연결해 중간 노드들 제거

# 4. 끝에 추가 (O(1))
lst.add(data)
# tail에서 바로 추가 (포인터가 있으니까 빠름)
```

## 막혔던 것 / 해결

**insert(0, nums) 처리의 복잡성:**
- idx=0 (맨 앞)일 때는 head를 새로 설정해야 하므로 특별 처리
- st=1로 해서 nums[1:]부터만 처리 (nums[0]은 head로 이미 처리)
- idx를 1로 바꾸는 이유: 그 다음부터는 일반적인 삽입 로직이 되도록

**remove(0, cnt) 처리:**
- idx=0 (맨 앞 삭제)일 때는 head를 cnt칸 뒤로 이동
- 일반 경우는 anchor.next = cur.next로 건너뛰기

**tail 관리:**
- insert/remove 후 마지막 노드가 바뀌었는지 확인
- cur.next == None이면 tail을 cur로 업데이트
- tail이 없으면 add()가 O(n)이 됨

## 참고

노드 풀 방식의 장점: 메모리 미리 할당으로 할당/해제 오버헤드 없음.
단점: 고정 크기 (nodeCnt > NODE_MAX 초과 시 문제).

삽입/삭제는 O(n) (위치 찾기 때문)이지만, 배열보다는 빠름.
특히 중간 삽입/삭제가 많을 때 유리.

SWEA 암호문 문제나 유사한 구조 변경 문제에서 자주 사용.
